# CS548 Assignment 6
## Noriel Valdes

The SOAP webservice for our pretend clinic is straightforward.  The Patient WS can create and retrieve patients, as well as retrieve information for treatments belonging to a particular patient.  The Provider WS can do all of the same thing for Providers, but can also create new treatments associated with a given Provider and Patient.

I believe the fact that I was able to successfully implement a tester that does not require an entity manager is sufficient proof that the SOAP service works to spec.

Appended below you will find PatientDTO.xsd, which describes the various types taken by the SOAP webservice, the generated WSDL for both PatientWebService and ProviderWebService, as well as the XSDs describing the arguments to for each function in the WSDL.
